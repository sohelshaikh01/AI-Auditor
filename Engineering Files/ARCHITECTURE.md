Why you chose your stack ?

**React (Frontend)**: React is library rather than a full framework, which allow you to hand-pick every tool in your stack from routing to state management rather than being locked into a specific way of doing things. React also used in creating complex UI and for it easy maintainability over `vanilla js`.
Also because of using Supabase for backend as a service, also that's why Next.js is not used.

**JavaScript**:
- Faster Development: As it does not require compilation steps
- Reduced Complexity: For small projects or simple utility scripts, it is best. Need no complex configurations like JavaScript.
- Lower Learning Curve: Beginner can focus on core programming concepts with focusing added type system.

**Supabase**: It provides easier setup for development and auto generated APIs.
- Making suitable for faster development and no need to deploy the backend.
